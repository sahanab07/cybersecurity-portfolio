# Splunk SIEM Investigation

## Overview

This project documents a Capture-the-Flag (CTF) style investigation completed using Splunk as part of the CodePath Intermediate Cybersecurity course.

The goal of this project was to practice using a Security Information and Event Management (SIEM) platform to search, analyze, and interpret indexed data. Using Splunk Search Processing Language (SPL), I investigated datasets, developed queries, and used search results to answer security-related challenges.

## Objectives

* Filter events using indexed fields
* Search specific datasets
* Count results with statistical commands
* Sort and identify trends in data
* Create targeted queries based on investigation goals

## Tools

* Splunk Enterprise
* Splunk Search Processing Language (SPL)
* Docker

## Skills

* Security Information and Event Management (SIEM)
* Log Analysis
* SPL Query Writing
* Data Filtering and Investigation
* Event Searching
* Threat Investigation Workflow

## Process

1. Accessed the Splunk lab environment and reviewed available datasets.
2. Used SPL queries to search and filter indexed data.
3. Applied fields and conditions to narrow investigation results.
4. Used statistical commands to analyze data patterns and identify answers.
5. Documented query approaches and findings during the investigation.

## Lessons Learned

This project improved my understanding of how SIEM platforms are used to investigate large amounts of data efficiently. I learned how to create structured SPL searches, analyze results, and approach investigations by forming questions and using available data to support conclusions.

## References

This project was completed as part of the CodePath Intermediate Cybersecurity course using a simulated SIEM investigation environment. The documentation in this repository reflects my own understanding of the investigation workflow and techniques practiced.
